# Docker Setup for FinanceApp

This document provides instructions for building and running the FinanceApp using Docker.

## Prerequisites

- Docker installed on your system
- Internet connection for pulling base images and dependencies

## Building the Docker Image

To build the Docker image, run the following command from the project root:

```bash
docker build -t finance-app .
```

This will:
1. Install all npm dependencies using `npm ci`
2. Build the Expo web application using `expo export --platform web`
3. Create a lightweight production image with the built assets
4. Configure the image to serve the app on port 3000

## Running the Container

To run the container:

```bash
docker run -p 3000:3000 finance-app
```

The application will be accessible at `http://localhost:3000`

### Running in Detached Mode

To run the container in the background:

```bash
docker run -d -p 3000:3000 --name finance-app-container finance-app
```

### Custom Port Mapping

To run the app on a different port (e.g., 8080):

```bash
docker run -p 8080:3000 finance-app
```

Then access the app at `http://localhost:8080`

## Managing Containers

### View running containers
```bash
docker ps
```

### Stop the container
```bash
docker stop finance-app-container
```

### Start the container again
```bash
docker start finance-app-container
```

### Remove the container
```bash
docker rm finance-app-container
```

### View container logs
```bash
docker logs finance-app-container
```

## Docker Image Details

- **Base Image**: node:22-alpine (lightweight Alpine Linux)
- **Build Strategy**: Multi-stage build (reduces final image size)
- **Web Server**: serve (lightweight static file server)
- **Port**: 3000
- **Output Directory**: dist/ (generated by `expo export`)

## Troubleshooting

### Build fails with npm errors
Try cleaning your local node_modules and rebuilding:
```bash
rm -rf node_modules package-lock.json
docker build --no-cache -t finance-app .
```

### Container won't start
Check the logs:
```bash
docker logs finance-app-container
```

### Port already in use
Either stop the service using port 3000 or use a different port mapping:
```bash
docker run -p 3001:3000 finance-app
```

## Files Included

- **Dockerfile**: Multi-stage build configuration for the Expo web app
- **.dockerignore**: Excludes unnecessary files from the build context (reduces build time and image size)
- **package.json**: Updated with `build` script for Expo web export

## Development vs Production

This Docker setup is configured for **production** use (serving the built web app). For development:

- Use `npm start` or `expo start` locally
- The Docker container serves only the built static files
- Hot reloading and development features are not available in the Docker container
